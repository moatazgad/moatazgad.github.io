# Project Descriptions (under 600 characters each)

## GenAI Platform Infrastructure

Designed and built the DevOps infrastructure for a GenAI platform enabling rapid deployment of AI-driven business capabilities. Architecture combines ECS for containerized workloads, API Gateway as the entry point fronting an ALB, Terraform for infrastructure provisioning, and Jenkins for CI/CD automation. HashiCorp Vault handles secrets management, Artifactory manages build artifacts, and Bitbucket drives source control workflows. Infrastructure is modular and reusable — new AI use cases deploy without rearchitecting the platform.

Skills: AWS, Terraform, Jenkins, HashiCorp Vault, ECS

## Securities Trading Platform

Owned the cloud infrastructure for a high-availability securities trading platform on Azure. Architecture includes AKS clusters spanning dev, test, and production, deployed via Terraform with Helm charts managing microservice configurations. Azure DevOps orchestrates CI/CD pipelines for automated build and release workflows. Elasticsearch clusters handle log management with data sharding, ILM and SLM policies; Kibana provides real-time observability across all environments.

Skills: Azure, AKS, Terraform, Helm, Elasticsearch

## AgriTech Cloud Infrastructure

Designed and implemented the deployment and infrastructure layer for a science-based agriculture platform on AWS. Architecture uses ECS for containerized workloads, GitLab CI/CD for pipeline automation with SonarCloud enforcing quality gates, and Terraform for infrastructure-as-code. AWS Batch handles scheduled data processing jobs. Linux servers are hardened and standardized. Artifacts are versioned and stored for full traceability and rollback capability across all environments.

Skills: AWS, GitLab CI/CD, Terraform, SonarCloud, Linux

## On-Prem AI & Automation Infrastructure

Deployed and tested LLM models on on-premises Linux servers for an intelligent sensing and power solutions company. Automated the full installation of Elasticsearch, Kibana, and RabbitMQ using Ansible playbooks. Configured multi-node Elasticsearch and RabbitMQ clusters for high availability and fault tolerance. Environment is fully on-prem — no cloud dependency. Infrastructure is repeatable, version-controlled, and designed for handoff to internal teams.

Skills: Linux, Ansible, Elasticsearch, RabbitMQ, LLM
