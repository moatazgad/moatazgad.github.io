On-Prem AI & Automation Infrastructure

Deployed LLM models on-premises and automated distributed infrastructure for an intelligent sensing and power solutions company building the future of cleaner energy, smarter mobility, and advanced automation.

Role DevOps & Systems Engineer
Type Client Engagement
Environment On-Premises
Linux Ansible Elasticsearch Kibana RabbitMQ LLM On-Prem

Overview

The client develops high-performance magnetic sensors and power ICs for next-generation applications in cleaner energy, smarter mobility, and advanced automation. They needed on-premises AI capabilities and robust messaging and observability infrastructure to support internal R&D and production workloads.

As the DevOps & Systems Engineer, I was responsible for deploying AI models locally, automating infrastructure provisioning, and ensuring the distributed systems were resilient and production-ready.

What I Built

  • Deployed and tested LLM models on on-premises Linux servers — enabling local AI inference without cloud dependency
  • Automated the full installation of Elasticsearch, Kibana, and RabbitMQ using Ansible playbooks — repeatable, idempotent, and version-controlled
  • Configured Elasticsearch in a multi-node cluster for high availability, data replication, and search performance
  • Set up RabbitMQ clustering for reliable distributed message queuing across services
  • Built Kibana dashboards for real-time monitoring and log visualization
  • Hardened Linux servers — managing access control, firewall rules, and system-level security

Results

On-Prem AI
LLM running locally with zero cloud dependency
Automated
Full stack provisioned via Ansible — minutes, not days
Clustered
Elasticsearch and RabbitMQ running in HA mode
Observable
Full visibility into system health via Kibana

Approach

This was a purely on-premises engagement — no cloud. Everything had to be automated, repeatable, and maintainable by the internal team after handoff. Ansible was the natural choice for configuration management: playbooks covered the full lifecycle from OS hardening to application clustering.

The LLM deployment required testing across different hardware configurations to ensure inference performance met requirements without cloud GPUs. Elasticsearch and RabbitMQ clusters were designed for fault tolerance — if a node goes down, the system keeps running.
