Overview

A securities trading platform requiring high availability, strict environment parity, and deep observability. The system needed to handle sensitive financial workloads with zero tolerance for downtime or data loss.

As the Cloud & DevOps Engineer, I managed the full infrastructure lifecycle — from provisioning clusters to deploying microservices to building the logging and monitoring stack.

What I Built

  • Managed AKS cluster deployments across development, test, and production environments
  • CI/CD pipelines using Azure DevOps for automated build, test, and release workflows
  • Infrastructure provisioning and lifecycle management with Terraform
  • Helm charts for deploying microservices and managing application configurations
  • Elasticsearch clusters for advanced log management — including data sharding, index management, ILM policies, and SLM policies
  • Kibana dashboards for real-time visibility into system health and application logs

Approach

The platform runs as a set of microservices on AKS. Each service is packaged with Helm, deployed through Azure DevOps pipelines, and monitored via Elasticsearch. Infrastructure changes go through Terraform — reviewed, planned, and applied with full audit trails.

The logging stack was designed for scale: data sharding keeps queries fast, ILM policies handle retention automatically, and SLM policies ensure backups are always current. No logs lost, no blind spots.
